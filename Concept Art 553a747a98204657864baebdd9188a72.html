Shape

Anahi Shape

O primeiro passo para a construção do personagem é trabalhar os elementos com formas básicas, construido a silhueta e desenvolvendo o shape do personagem.

Após o desenvolvimento desse processo começamos a refinar o shape e modelar o personagem, pra isso será utilizado o processo de blocagem, que consistem em definir as formas e características básicas como braços, pernas, troncos.

Nessa etapa você começa a detalhar melhor o personagem para ter a estrutura do personagem.

Sprite Sheet

A construção do sprite-sheet tem uma etapa anterior que é bem importante definir o tipo de animação que será realizada, pois isso vai ser bem importante na construção do personagem. Se for uma animação tradicional frame a frame, o processo consiste em desenhar a pose inicial do personagem e a cada frame é realizado um novo desenho fazendo pequenas modificações do personagem e sua pose ou mesmo posição.

Se a animação for realizada utilizando a interpolação de movimento a recomendação é que o concept do personagem seja pensado de outra forma, o recomentado é que as partes do corpo ou mesmo as roupas sejam planejadas em separado pois isso vai facilitar no proceso de animação principalmente para um modelo 2D.

Dito isso esse primeiro modelo desenvolvido foram desenhados apenas os quadros chaves já na posição não sendo feito esse processo de separação de membros dessa maneira a construção dos sprites foram pensada de forma a proporcionar um movimento de corrida, com uma menor quantidades de quadros para o desenvolvimento da animação.

Nesse sprite sheet, as poses não foram criadas utilizando os bones(Esqueleto de animação), que é uma outra técnica que ajuda a construir animações, porém na utilização dessa técnica é necessário a construção das partes separadas.

Aqui eu preciso colocar uma observação importante o trabalho de animação tradicional é um universo particular, que demanda todo um estudo mais aprofundado a respeito como os 12 passos da animação. Para o desenvolvimento de jogos principalmente em 2D já conseguimos trabalhar com essa estrutura simples de animação pensando nos quadros chaves para dar a sensação de movimento, no jogo.

Uma questão importante na construção do sprite é o seu tamanho, pois quando ele é implementado no jogo essa questão é muito importante para que possa ser configurado na engine os cortes caso seja utilizado apenas uma folha de Sprite para os assets, que é chamado de atlas.

Folha de Sprite para implementação no jogo.

Essa foi a folha de sprite aplicada no jogo a caçada do Uirapuru do projeto de Smaug do 2º Ciclo projeto desenvolvido utilizando o python e pygame como engine.

Como o projeto não tinha um tempo muito extenso para a sua produção, optamos por uma linguagem mais simplificada onde fosse possível a construção de assets de todo o jogo, além disso a equipe ainda não tinha recursos, de tempo e conhecimentos necessários para produzir uma animação mais elaborada, pois essa construção é feita em conjunto pois diferente da animação tradicional, para jogos precisamos levar em consideração a parte de programação e como o personagem vai se mover no jogo.

Dessa maneira, para entregar o projeto fizemos uma quantidade menos de sprites facilitando o processo de animação para o jogo, além disso outra característica importante é que como o concept do personagem não é simétrico, foi necessário pensar no sprite conforme a sua movimentação no jogo, sendo necessário fazer a animação de corrida para a direita e esquerda assim como as animações de pulo em ambos os sentidos, pois em alguns casos se a construção dos sprites for simétrica, é possivel reaproveitar a animação dentro da engine apenas com o flip. Em projetos de baixo orçamento ou mesmo com uma demanda de tempo curta essa é uma prática comum.

Estados de animação

Uma característica importante que eu só consegui compreender depois são os estados da animação pensando em jogos.

Essas animações são pensadas de acordo com as características que você quer passar para o personagem e as ações dele de acordo com o game design do jogo.

Algumas animações mais simples que podemos pensar para iniciar o projeto podem ser a animação de Idle (Parado), Walk (Andando), Run (Correndo), jump (Pulando).

Essas são algumas animações simples que já podem auxiliar em um jogo no estilo platform por exemplo.

Porém é completamente possível construir um jogo simples com apenas um estado da animação, o jogo que foi desenvolvido no primeiro semestre para o projeto de Smaug, eu utilizei apenas 2 quadros chaves para a animação do rôbo, com o estado de ligado e desligado.
