Após o desenvolvimento desse processo começamos a refinar o shape e modelar o personagem, pra isso será utilizado o processo de blocagem, que consistem em definir as formas e características básicas como braços, pernas, troncos. </p><p id="7657baee-feb5-4e47-ab0b-0d75c80f90d9" class="">Nessa etapa você começa a detalhar melhor o personagem para ter a estrutura do personagem. </p></div></div><h2 id="1edb0075-8748-4b6b-bfae-11d1c593e460" class="">Sprite Sheet</h2><p id="d6bea4c6-d610-4f58-a63d-edb86d0354a7" class="">A construção do sprite-sheet tem uma etapa anterior que é bem importante definir o tipo de animação que será realizada, pois isso vai ser bem importante na construção do personagem. Se for uma animação tradicional frame a frame, o processo consiste em desenhar a pose inicial do personagem e a cada frame é realizado um novo desenho fazendo pequenas modificações do personagem e sua pose ou mesmo posição. </p><p id="50b0293f-d381-4cb1-a64b-736974bca53b" class="">Se a animação for realizada utilizando a interpolação de movimento a recomendação é que o concept do personagem seja pensado de outra forma, o recomentado é que as partes do corpo ou mesmo as roupas sejam planejadas em separado pois isso vai facilitar no proceso de animação principalmente para um modelo 2D. </p><p id="6fcd46b7-a89c-4c64-9161-18973d1c133e" class="">Dito isso esse primeiro modelo desenvolvido foram desenhados apenas os quadros chaves já na posição não sendo feito esse processo de separação de membros dessa maneira a construção dos sprites foram pensada de forma a proporcionar um movimento de corrida, com uma menor quantidades de quadros para o desenvolvimento da animação. </p><p id="86de695f-7190-4d93-be0d-1669b2f7fa01" class="">Nesse sprite sheet, as poses não foram criadas utilizando os bones(Esqueleto de animação), que é uma outra técnica que ajuda a construir animações, porém na utilização dessa técnica é necessário a construção das partes separadas. </p><p id="111ffd45-2a7e-42d5-b019-f2c0c0ef4808" class="">
</p><p id="e29fde42-fb16-415f-8a93-34b4d0f8571e" class="">Aqui eu preciso colocar uma observação importante o trabalho de animação tradicional é um universo particular, que demanda todo um estudo mais aprofundado a respeito como os 12 passos da animação. 

Para o desenvolvimento de jogos principalmente em 2D já conseguimos trabalhar com essa estrutura simples de animação pensando nos quadros chaves para dar a sensação de movimento, no jogo. </p><p id="98245f45-cbd2-403d-8399-3a566872ba4f" class="">Uma questão importante na construção do sprite é o seu tamanho, pois quando ele é implementado no jogo essa questão é muito importante para que possa ser configurado na engine os cortes caso seja utilizado apenas uma folha de Sprite para os assets, que é chamado de<strong> atlas. </strong></p><h3 id="b727b4ee-4a35-4684-9400-2c00e6319254" class="">Sprite-sheet 01 Corrida</h3><figure id="0196a82c-e289-4be9-98e2-c7b3e92f6f67" class="image"><a href="Concept%20Art%20553a747a98204657864baebdd9188a72/sprite-sheet---player-1.png"><img style="width:1200px" src="Concept%20Art%20553a747a98204657864baebdd9188a72/sprite-sheet---player-1.png"/></a></figure><h3 id="078570f3-65e3-4e9a-9dc3-f63bceb23cd3" class="">Sprite-sheet 02 Pulo</h3><figure id="3794b353-f7b9-4a91-9896-9efc1cda6b05" class="image"><a href="Concept%20Art%20553a747a98204657864baebdd9188a72/sprite-sheet-pulo-128-bits.png"><img style="width:895px" src="Concept%20Art%20553a747a98204657864baebdd9188a72/sprite-sheet-pulo-128-bits.png"/></a></figure><p id="e09da21f-5ee2-4f51-a0c7-d99b18782f25" class="">
</p><h3 id="d51ffa8b-af95-4e52-9fd6-029cd85e6789" class="">Folha de Sprite para implementação no jogo. </h3><div id="8ccaee89-771a-4d39-aa36-429002dc0265" class="column-list"><div id="254fd0f0-6aa9-469d-a96f-7827b90d04b0" style="width:100%" class="column"><figure id="25af5d05-a10e-46ce-a27b-7ead457e286d" class="image"><a href="Concept%20Art%20553a747a98204657864baebdd9188a72/player2.png"><img style="width:624px" src="Concept%20Art%20553a747a98204657864baebdd9188a72/player2.png"/></a></figure></div><div id="2dc709d4-920d-47dd-b774-4f3bc7892bca" style="width:100%" class="column"><p id="54b69d1c-0418-4dec-a412-c94a356cfbc8" class="">Essa foi a folha de sprite aplicada no jogo a caçada do Uirapuru do projeto de Smaug do 2º Ciclo projeto desenvolvido utilizando o python e pygame como engine.</p><p id="2fc49a25-141d-4758-8f5c-a5fd478f3402" class="">Como o projeto não tinha um tempo muito extenso para a sua produção, optamos por uma linguagem mais simplificada onde fosse possível a construção de assets de todo o jogo, além disso a equipe ainda não tinha recursos, de tempo e conhecimentos necessários para produzir uma animação mais elaborada, pois essa construção é feita em conjunto pois diferente da animação tradicional, para jogos precisamos levar em consideração a parte de programação e como o personagem vai se mover no jogo. </p><p id="27750efe-0e3d-4dbd-ab75-bb6645de7812" class="">Dessa maneira, para entregar o projeto fizemos uma quantidade menos de sprites facilitando o processo de animação para o jogo, além disso outra característica importante é que como o concept do personagem não é simétrico, foi necessário pensar no sprite conforme a sua movimentação no jogo, sendo necessário fazer a animação de corrida para a direita e esquerda assim como as animações de pulo em ambos os sentidos, pois em alguns casos se a construção dos sprites for simétrica, é possivel reaproveitar a animação dentro da engine apenas com o flip. Em projetos de baixo orçamento ou mesmo com uma demanda de tempo curta essa é uma prática comum. </p></div></div><h3 id="cd19db78-fc4b-4b20-91b3-abe4a772c16f" class="">Estados de animação</h3><p id="e051c1d6-c94a-49ee-99bf-610126c6311a" class="">Uma característica importante que eu só consegui compreender depois são os estados da animação pensando em jogos. </p><p id="990de02b-1450-4509-b92e-bc1f2eee7a69" class="">Essas animações são pensadas de acordo com as características que você quer passar para o personagem e as ações dele de acordo com o game design do jogo. </p><p id="13c23e76-036e-4cd1-b2a4-f258e4ef0685" class="">Algumas animações mais simples que podemos pensar para iniciar o projeto podem ser a animação de Idle (Parado), Walk (Andando), Run (Correndo), jump (Pulando).</p><p id="f610b6b7-04a4-40d1-9b1b-2a8cce5fcc87" class="">Essas são algumas animações simples que já podem auxiliar em um jogo no estilo platform por exemplo. </p><div id="96d90fed-26a0-41e8-8702-222d0b3ff5e3" class="column-list"><div id="3b58d6fc-99a7-466e-8169-6a1ac22a0138" style="width:40.625%" class="column"><p id="f649302b-b51e-42f6-af34-b6344ed1d6f6" class="">Porém é completamente possível construir um jogo simples com apenas um estado da animação, o jogo que foi desenvolvido no primeiro semestre para o projeto de Smaug, eu utilizei apenas 2 quadros chaves para a animação do rôbo, com o estado de ligado e desligado. </p><p id="7cdee504-100b-49c8-bf7e-eeef8cd0ce58" class="">
